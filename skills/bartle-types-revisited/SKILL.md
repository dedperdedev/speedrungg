# Bartle Types — Revisited

Richard Bartle's MUD player taxonomy (1996). Useful starting point, but oversimplified — read with critique.

## The four

| Type | Wants | Verb |
|---|---|---|
| **Achiever** | Mastery, completion, status | "I want to win" |
| **Explorer** | Discovery, knowledge, system understanding | "I want to know" |
| **Socializer** | Connection, communication, belonging | "I want to be with" |
| **Killer** | Dominance, impact on others | "I want to be feared" |

## What Bartle got right

- Players have stable preferences across games
- Games designed for one type often alienate others
- Mixed populations create emergent dynamics

## What Bartle got wrong (or oversimplified)

- The 4 types are not exhaustive (no Casual / Maintenance / Cozy type — added in later models)
- Not all players are stable (people switch types based on mood, life context)
- "Killer" is loaded — most actually want **agency over outcome**, not domination
- The model assumes synchronous multiplayer (was built for MUDs)
- Demographic / cultural variation is huge

## Updated 6-type model (more useful in 2026)

For most web games:

| Type | Description |
|---|---|
| Achiever | Mastery + completion |
| Explorer | Discovery + system mastery |
| Socializer | Connection + communication |
| Competitor | Skill expression vs others (replaces "Killer") |
| Creator | Building / customizing / sharing creations |
| Cozy | Low-pressure, autonomous-paced experience |

## Use in design

Don't ask "is my game for Achievers or Explorers?" That's binary thinking.

Ask:
1. Which 1–2 types does this game **primarily** serve?
2. Which types does it accidentally **exclude** through design choices?
3. Which type would HATE this game, and is that intentional?

A good design has a clear primary type + one secondary + intentional exclusion of incompatible types.

## Anti-patterns

- "All types welcome!" → no design focus, no one is served
- Designing for "the typical gamer" → that person doesn't exist
- Adding leaderboards (Competitor) to a Cozy game without a toggle
- Adding open-world exploration (Explorer) to a Hard-fun game and expecting both audiences
